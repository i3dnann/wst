# World Star complete UI redesign prompt

Redesign the existing World Star React website and protected administrator application as one responsive product system. Preserve every real route, API workflow, permission check, CRUD action, bracket workflow, query state, and administrator capability. Do not replace working functionality with mock data.

Use the supplied metallic crimson WST logo as the only primary brand mark. The visual language is modern mafia command-center: true black page canvas, graphite and charcoal surfaces, crimson red interaction color, cool white primary text, and neutral gray secondary text. Remove every yellow, gold, beige, cyan, purple, and orange treatment, including text, loading states, panels, borders, gradients, focus rings, badges, charts, and button highlights.

Use a wide condensed display font with comfortable positive letter spacing for headings and a highly legible sans serif for body copy, inputs, tables, navigation, and buttons. No heading may look cramped, clipped, or overlap at any screen size.

The public header must use a full-width background from left edge to right edge, a readable logo lockup, essential routes, one admin action, a clear active state, and a mobile drawer. Rebuild the home hero around the logo with a restrained red Magic UI ripple background and a red Magic UI shimmer primary action. Hero copy: “Where every rivalry becomes history. Follow verified matches, live tournaments, gang rankings, events, and streams from one official command center.” Continue with real tournament, event, gang-ranking, and live-stream data in responsive rails, timelines, lists, and tables.

Build buttons with 44–48px minimum height, clipped-corner geometry, clear hover/pressed/focus/disabled states, a restrained edge shimmer on primary actions, and distinct neutral, danger, and ghost variants. Build inputs, textareas, selects, dropdowns, and popovers with visible labels, generous padding, clear red focus rings, consistent option surfaces, readable error states, and strong keyboard accessibility.

Use 21st.dev-inspired spotlight panels and responsive dashboard composition, but avoid nested card clutter. Cards and panels use thin neutral borders, subtle red spotlight illumination, no heavy shadows, consistent internal spacing, and layouts that adapt from wide desktop through tablet to small mobile screens.

Redesign every public route—gangs, gang detail, players, player detail, tournaments, tournament detail, brackets, matches, match detail, rankings, events, live, rules, about, maintenance, login, empty states, error states, and loading states—inside this shared system.

Redesign the admin as a persistent responsive command center. Group the navigation into Overview; Registry; Competition; Publishing; Access & System. Keep the rail visible on desktop and provide a compact always-available navigation surface on smaller screens. Use a compact top header, operational metric tiles, refined record tables, focused editor drawers, confirmation dialogs, filters, selects, media tools, and settings panels. Preserve permission-aware route visibility and all existing add/edit/delete/archive/bracket workflows.

The administrator must be able to manage the site name, short name, logo URL, favicon, description, maintenance mode, hero title/subtitle/media, announcement, three red-safe brand colors, animation intensity, tournament defaults, registration rules, social links, Rules page title/introduction/content, and About page title/introduction/content. Stored legacy gold defaults should be normalized to the new crimson defaults without removing custom administrator choices.

Use React, TypeScript, the existing routing/query/form stack, Magic UI shimmer button and ripple patterns, reusable components, CSS custom-property tokens, accessible semantic HTML, reduced-motion support, and responsive layouts. Verify typecheck, lint, tests, production build, desktop rendering, mobile rendering, navigation, a primary public action, and an administrator interaction before approval.
